# 价格行为学交易筛查系统 — 规则与问题树

## 设计理念
> 不是帮交易者看盘分析，而是帮交易者想清楚。
> 交易者在下单前，通过回答一系列基于价格行为学的问题，
> 系统自动筛查这笔交易满足哪些高胜率条件、违反了哪些铁律，
> 最终给出综合评估，辅助交易者判断是否值得下单。

---

## 一、筛查流程总览

```
标的信息输入
  → A. 市场环境识别 (Context)
    → B. 交易方向确认 (Direction)
      → C. 入场信号检查 (Entry)
        → D. 风控检查 (Risk)
          → E. 目标位检查 (Target)
            → F. 综合评分与建议 (Assessment)
```

---

## 二、各阶段问题树

### A. 市场环境识别 (Context) — "你交易的环境是什么？"

**核心铁律：Context永远大于信号K**

A1. 当前市场处于哪个周期阶段？
  - 突破 (Breakout/Spike): 连续大实体K线，几乎无回调
  - 窄通道 (Tight Channel): 单边趋势，回调仅1-2根K线
  - 宽通道 (Broad Channel): 趋势方向延续但K线重叠多、回调深
  - 震荡区间 (Trading Range): 大涨大跌交替，无一方占优

  → 分支逻辑：
    - 选"突破"或"窄通道" → 进入强趋势分支
    - 选"宽通道" → 进入弱趋势分支
    - 选"震荡区间" → 进入震荡分支

A2. 价格目前在EMA20上方还是下方？
  - 上方 → 多头背景
  - 下方 → 空头背景
  - 附近纠缠 → 模糊区域

A3. （强趋势分支追加）
  A3a. 这是窄通道还是宽通道？
  A3b. 窄通道中，有均线缺口吗？（连续N根K线未触碰EMA20）
  A3c. 窄通道运行了多久？（第一段/第二段/第三段+）

A4. （震荡分支追加）
  A4a. 区间宽度大约是多少点？
  A4b. 当前价格在区间的什么位置？（上沿1/3 / 中间1/3 / 下沿1/3）

A5. （宽通道分支追加）
  A5a. K线重叠度如何？（高重叠/中等重叠/低重叠）
  A5b. 宽通道运行了几推？（第一推/第二推/第三推）

---

### B. 交易方向确认 (Direction) — "你的方向和背景一致吗？"

B1. 你打算做多还是做空？

B2. 这是顺势交易还是逆势交易？
  - 顺势 → 方向与EMA20/大趋势一致
  - 逆势 → 方向与EMA20/大趋势相反

  → 分支逻辑：
    - 顺势 + 强趋势背景 → 走"顺势强趋势"路径
    - 顺势 + 弱趋势背景 → 走"顺势弱趋势"路径
    - 顺势 + 震荡背景 → 赜"区间内高抛低吸"路径
    - 逆势 + 强趋势背景 → 🔴 高危！进入强警告
    - 逆势 + 弱趋势/震荡 → 走"逆势反转"路径

**🔴 致命错误检测：**

RED-1: 在窄通道/突破中逆势交易？
  → "第一次反转80%概率失败，这是极高风险行为"

RED-2: 在震荡区间边缘打算追涨/追空（第二段）？
  → "震荡区间第二段是陷阱，不加仓不开仓"

RED-3: 在震荡区间用突破单(Stop Order)追突破？
  → "80%突破会失败，震荡区间应使用限价单(Limit Order)"

---

### C. 入场信号检查 (Entry) — "你的入场理由是什么？"

C1. 你看到了什么信号打算入场？
  - 一根信号K线（趋势K线收在极端）
  - 双K线反转
  - 多K线组合反转
  - 结构形态（双顶/双底/楔形）
  - 数K线计数（H1/H2/H3 或 L1/L2/L3）
  - EMA20回踩
  - 突破某个关键位
  - 其他/纯直觉

C2. 信号K线评分：
  C2a. 信号K线的实体大小？（相对于近期平均K线）
    - 远大于平均 (+3分)
    - 略大于平均 (+1分)
    - 与平均相当 (0分)
    - 小于平均 (-1分)

  C2b. 信号K线收盘位置？
    - 收在最极端（最高/最低点附近） (+3分)
    - 收在实体2/3以上 (+1分)
    - 收在中间 (0分)
    - 收在1/3以下 (-1分)

  C2c. 信号K线是否有长影线向你的方向？
    - 是，长下影线做多/长上影线做空 (+2分，代表逆势方尝试失败)
    - 否 (0分)

  C2d. 信号K线是趋势K还是震荡K？
    - 趋势K（实体占比>60%） (+2分)
    - 震荡K（十字星/影线占比大） (-1分)

C3. 数K线计数：
  C3a. 这是第几次入场信号？
    - H1/L1 → 第一次入场（⚠️ 80%可能失败，胜率低）
    - H2/L2 → 第二次入场（✅ 黄金入场点，胜率最高）
    - H3/L3 → 第三次入场（✅ 楔形回调，好楔形75%胜率）
    - H4/L4+ → 第四次及以上（⚠️ 趋势极弱，可能已转震荡）

C4. 楔形质量评估（如果看到楔形结构）：
  C4a. K线重叠度？
    - 高重叠（好楔形，胜率可达75%） → ✅
    - 低重叠/窄通道式（坏楔形/假楔形） → 🔴 "这不是楔形，是窄通道，反转胜率仅20%"

  C4b. 这是反转楔形还是中继楔形旗形？
    - 顺大势逆小势的中继旗形 → ✅ 胜率更高
    - 逆大势的反转楔形 → ⚠️ 需要更强的确认

C5. 二次入场检查：
  C5a. 你是第一次入场还是第二次入场？
    - 第一次 → ⚠️ "强趋势第一次反转80%失败，建议等待二次入场"
    - 第二次 → ✅ "二次入场是高阶高胜率武器"

C6. 你的订单类型是什么？
  - Stop Order（突破单） → 在信号K高点/低点外1tick挂单 ✅
  - Limit Order（限价单） → 在支撑/阻力区挂单
  - Market Order（市价单） → ⚠️ 大部分情况不建议

  → 分支逻辑：
    - 震荡区间 + Stop Order → 🔴 "震荡区间80%突破失败，应改用Limit Order"
    - 强趋势 + Market Order → ✅ "Always In行情可以用市价"
    - 一般情况 + Market Order → ⚠️ "建议用Stop Order过滤信号"

---

### D. 风控检查 (Risk) — "你的止损合理吗？"

D1. 你有保护性止损吗？
  - 有，已经设定好 → 继续
  - 有，但只是"心理止损" → 🔴 "心理止损等于没有止损！必须挂在系统里"
  - 没有 → 🔴🔴 "开仓后第一件事是设止损，不允许无止损交易"

D2. 止损放在哪里？
  - 信号K线极端点外1tick → ✅ 最规范
  - 前低/前高外1tick → ✅ 结构性止损
  - 整体起涨/起跌点外 → ✅ 大止损但合理（实际风险可能很小）
  - 固定点数/百分比 → ⚠️ 不够灵活
  - "大概在那里" → 🔴 "止损位必须精确，不能大概"

D3. 止损订单类型？
  - Stop Market → ✅ 确保100%执行
  - Stop Limit → 🔴 "可能滑过不成交，失去保护意义，只用Stop Market"

D4. 臭止损检测：
  D4a. 如果做多，止损放在震荡区间下沿1/3区域吗？
    → 🔴 "这是臭止损！别人在这里买入，你在这里卖出"

  D4b. 如果做空，止损放在震荡区间上沿1/3区域吗？
    → 🔴 "这是臭止损！80%突破会失败，别人在这里做空"

D5. 单笔风险占账户比例？
  - ≤1% → ✅ 完美风控
  - 1-2% → ✅ 合理
  - 2-5% → ⚠️ 偏高
  - >5% → 🔴 "风险过大，即使好信号也不值得冒这么大风险"

D6. 入场前提条件是什么？
  → 提醒："如果前提条件被打破（如震荡区下沿被跌破），必须无条件离场，不论盈亏"

---

### E. 目标位检查 (Target) — "你的目标位有依据吗？"

E1. 你的止盈目标位是什么？
  - 有明确目标位 → 继续
  - 没有 → ⚠️ "没有目标位的交易容易拿不住或贪心"

E2. 目标位的依据？
  - Measured Move（区间翻倍） → ✅
  - Leg1=Leg2等长测算 → ✅
  - 信号K实体翻倍 → ✅
  - 前期关键高/低点 → ✅
  - 2倍止损距离（2R） → ✅
  - Actual Risk的1-2倍 → ✅✅ 最精确
  - "感觉差不多" → ⚠️ "目标位需要客观测算依据"

E3. 盈亏比？
  - ≥2:1 → ✅ 满足40%胜率+2R的稳定盈利条件
  - 1:1 → ⚠️ 需要60%+胜率才能盈利
  - <1:1 → 🔴 "盈亏比不合理，即使赢了也不划算"

E4. 分批止盈计划？
  - 有（1R先止盈一半，剩余看2R+） → ✅✅ Smart Trade策略
  - 没有，打算一次全出 → ⚠️ "建议分批止盈锁定胜率"

E5. 第三推后打算怎么办？
  - 减仓/止盈一部分 → ✅
  - 继续全仓持有 → ⚠️ "第三推后大概率反转或震荡"

E6. 如果是楔形反转交易，是否知道失败后的应对？
  - 知道 → ✅ "楔形失败后有一倍至两倍测量运动，必须无条件止损"
  - 不知道 → ⚠️ "每个形态都有25%失败概率，必须有离场预案"

### F. 综合评分与建议 (Assessment)

#### 评分维度与权重

| 维度 | 权重 | 说明 |
|---|---|---|
| Context匹配度 | 30% | 方向是否与市场周期一致，最核心 |
| 信号质量 | 25% | 信号K评分 + 计数K线等级 |
| 入场方式 | 15% | 订单类型、入场次数是否正确 |
| 风控合理性 | 15% | 止损位置、订单类型、风险比例 |
| 目标位与盈亏比 | 15% | 目标依据、盈亏比、分批止盈 |

#### 综合评级

| 总分 | 评级 | 建议 |
|---|---|---|
| 85-100 | ✅✅ 优质交易 | 满足大部分高胜率条件，可以执行，建议用OCO订单Set and Forget |
| 70-84 | ✅ 合格交易 | 有一些瑕疵但核心条件满足，可以执行但需格外注意瑕疵项 |
| 50-69 | ⚠️ 存疑交易 | 多个条件不理想，建议缩小仓位或等待更好的入场点 |
| 0-49 | 🔴 不建议交易 | 违反铁律或核心条件缺失，强烈建议放弃这笔交易 |

#### 交易类型标签

系统会为这笔交易贴上类型标签，帮助交易者理解：
- "顺势强趋势H2入场" — 最稳健
- "震荡区间限价单高抛低吸" — 赌突破失败
- "三推楔形二次入场反转" — 高胜率逆势
- "窄通道逆势摸顶/抄底" — 🔴 致命错误

## 三、🔴 致命错误清单（立即阻止交易）

以下任何一条成立，系统直接给出红色警告，建议放弃交易：

1. **窄通道/突破中逆势** → "第一次反转80%概率失败"
2. **震荡区间边缘第二段追涨/追空** → "第二段陷阱"
3. **没有保护性止损** → "不允许无止损交易"
4. **心理止损而非系统止损** → "心理止损等于没有止损"
5. **止损用Stop Limit而非Stop Market** → "可能滑过不成交"
6. **单笔风险>5%账户** → "风险不可接受"
7. **震荡区间用Stop Order追突破** → "80%突破会失败"
8. **坏楔形（窄通道式无重叠）逆势摸顶/抄底** → "这不是楔形，是窄通道，反转胜率仅20%"
9. **在震荡区下沿1/3做多的止损放在下沿1/3** → "臭止损"
10. **在震荡区上沿1/3做空的止损放在上沿1/3** → "臭止损"

## 四、✅ 高胜率加分项清单

以下每满足一条，为交易加分：

1. 顺势交易（方向与EMA20和市场周期一致）(+10分)
2. H2/L2入场（二次入场，黄金入场点）(+15分)
3. H3/L3入场 + 高重叠好楔形（75%胜率）(+10分)
4. 信号K：大实体、收在极端、趋势K (+5分每项)
5. 信号K有长影线向你的方向（逆势方尝试失败）(+5分)
6. 使用正确的订单类型（顺势→Stop，震荡→Limit）(+10分)
7. 嵌套楔形/多重形态共振(+10分)
8. 顺大逆小中继旗形(+10分)
9. 止损在信号K极端点外1tick(+5分)
10. 单笔风险≤1%(+5分)
11. 有Measured Move等客观目标位(+5分)
12. 盈亏比≥2:1(+10分)
13. 分批止盈计划（Smart Trade）(+5分)
14. OCO订单（沃尔玛交易法）(+5分)
15. 多周期共振确认(+5分)
16. 缺口存在且未被回补（趋势强势信号）(+5分)
17. 50%回调位 + 信号K双重确认(+5分)

---

## 五、⚠️ 瑕疵减分项清单

以下每存在一条，扣分：

1. H1/L1入场而非H2/L2 (-10分)
2. 信号K实体较小 (-5分)
3. 信号K收盘不在极端 (-5分)
4. 震荡K线作为信号K (-8分)
5. 第一次入场而非二次入场 (-8分)
6. 使用市价单而非Stop Order（非Always In行情）(-5分)
7. 宽通道中追高/追低 (-10分)
8. 止损位置不够精确 (-5分)
9. 没有明确目标位 (-8分)
10. 盈亏比<2:1 (-10分)
11. 没有分批止盈计划 (-5分)
12. 前提条件容易被打破 (-5分)

## 六、交互流程设计

### 输入阶段
交易者输入：
- 标的名称/代码（如 ES、BTC、AAPL）
- 交易方向（做多/做空）
- 当前价格（辅助计算）
- EMA20位置（自己从图表上看）
- 其他基本参数

### 问答阶段
系统按照A→B→C→D→E的顺序逐项提问。
- 每个问题都有明确的选项，交易者选择最符合当前盘面的。
- 根据选择动态调整后续问题（分支逻辑）。
- 随时检测致命错误，一旦触发立即弹出红色警告。

### 输出阶段
系统输出：
1. 各维度评分明细
2. 满足的高胜率条件清单（绿色）
3. 存在的瑕疵清单（黄色）
4. 触发的致命错误（红色）
5. 综合评级和建议
6. 交易类型标签
7. 具体的操作建议（止损位、目标位、订单类型）
