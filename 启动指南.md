- Python 3.8+
- 依赖包：fastapi, uvicorn, jinja2

## 三、Web 界面使用流程

### 第1步：填写交易基本信息

输入标的代码（如 ES、BTC、AAPL），选择做多/做空，填写当前价格。

### 第2步：识别市场环境

从你的K线图上判断当前市场周期：

- **突破**：连续大实体K线，几乎无回调
- **窄通道**：单边趋势，回调仅1-2根K线
- **宽通道**：趋势延续但K线重叠多、回调深
- **震荡区间**：大涨大跌交替，无一方占优

同时标注价格在EMA20的上方/下方。

### 第3步：确认交易方向

你是顺势还是逆势？这是最关键的判断之一。

### 第4步：检查入场信号

- 你看到了什么信号（信号K、双K线反转、结构形态等）
- 信号K质量评分（0-100）
- 第几次入场（H1/L1、H2/L2、H3/L3）
- 楔形重叠度
- 订单类型（Stop/Limit/Market）

### 第5步：风控检查

- 是否有止损、止损方式、止损订单类型
- 臭止损检测
- 单笔风险比例

### 第6步：目标位与盈亏比

- 有无目标位及依据
- 盈亏比
- 是否分批止盈
- 第三推应对、失败预案

### 第7步：查看结果

点击"开始筛查"后，系统输出：

- **综合评分**（0-100）与评级
- **致命错误**（红色，必须解决）
- **满足的高胜率条件**（绿色）
- **瑕疵与警告**（黄色）
- **严重问题**（红色）
- **操作建议**
- **数学期望检查**

## 四、评分体系

| 分数 | 评级 | 含义 |
|------|------|------|
| 85-100 | ✅✅ 优质交易 | 满足大部分高胜率条件，用OCO订单Set and Forget |
| 70-84 | ✅ 合格交易 | 核心条件满足，注意瑕疵项，必要时缩小仓位 |
| 50-69 | ⚠️ 存疑交易 | 缩小仓位至1/3，或等待更好入场点 |
| 0-49 | 🔴 不建议交易 | 违反铁律，建议放弃 |

## 五、10条致命错误（自动检测）

触发任意一条即给出红色警告：

1. 窄通道/突破中逆势交易 — 第一次反转80%失败
2. 震荡区间边缘第二段追涨/追空 — 第二段陷阱
3. 没有保护性止损 — 不允许无止损交易
4. 心理止损而非系统止损 — 心理止损等于没有止损
5. 止损用Stop Limit而非Stop Market — 可能滑过不成交
6. 单笔风险超过5% — 风险不可接受
7. 震荡区间用Stop Order追突破 — 80%突破失败
8. 坏楔形（窄通道式无重叠）逆势 — 反转胜率仅20%
9. 做多止损放在区间下沿1/3 — 臭止损
10. 做空止损放在区间上沿1/3 — 臭止损

## 七、核心铁律速查

交易前在心中默念：

1. **Context永远大于信号K** — 同一根K线在不同环境下含义截然相反
2. **80/20规则** — 震荡中80%突破失败，趋势中80%反转失败
3. **不看到信号K不入场** — 用Stop Order过滤，不追收盘
4. **二次入场优于一次** — H2/L2是黄金入场点
5. **止损必须挂在系统里** — 心理止损等于没有止损
6. **三推后必须减仓** — 第三推进出后大概率反转或震荡
7. **开仓后第一件事设止损** — 不是分析下一笔交易

## 八、AI读图模式

### 功能说明

上传K线截图，AI根据Al Brooks价格行为学课程笔记自动分析，识别高胜率交易机会。

### 截图要求

| 要求 | 说明 |
|------|------|
| 最少K线数 | 60根（AI需要看到完整市场周期结构） |
| 推荐K线数 | 100~200根（最佳分析效果） |
| EMA20 | 截图中必须显示EMA20均线 |
| 时间级别 | 上传时标注（5min/15min/1h/4h/日线/周线） |
| 多周期 | 建议上传2~3个时间级别（如日线+4小时+15分钟） |
| 清晰度 | K线实体和影线清晰可见，避免模糊或截断 |
| 推荐平台 | TradingView（保留完整价格标尺） |

### 支持的AI模型

| 模型 | 环境变量 | 说明 |
|------|----------|------|
| OpenAI GPT-4o | `OPENAI_API_KEY` | 推荐首选，视觉分析能力强 |
| OpenAI GPT-4.1 | `OPENAI_API_KEY` + `OPENAI_MODEL=gpt-4.1` | 新模型，性价比高 |
| Claude Sonnet | `ANTHROPIC_API_KEY` + `AI_PROVIDER=claude` | 视觉分析能力强 |
| 智谱 GLM-4V | `ZHIPU_API_KEY` + `AI_PROVIDER=zhipu` | 国产模型 |
| 通义千问 Qwen-VL | `DASHSCOPE_API_KEY` + `AI_PROVIDER=qwen` | 国产模型 |
| DeepSeek | `DEEPSEEK_API_KEY` + `AI_PROVIDER=deepseek` | 国产模型 |

### 使用流程

1. 访问 http://127.0.0.1:8000/analyze
2. 输入标的代码
3. 选择AI模型
4. 上传K线截图（1~3张），标注时间级别
5. 点击"开始AI分析"
6. 等待30~60秒，查看分析结果

### 分析结果说明

- **市场概览**：当前市场周期阶段、EMA20位置、趋势方向、均线缺口、推数
- **交易机会**：每个机会包含方向、入场类型、信号K质量、止损/目标位、盈亏比、置信度
- **风险警告**：需要注意的风险点
- **AI原始返回**：可折叠查看完整分析文本

## 十、Vercel 部署

### 部署步骤

1. 将项目推送到 GitHub 仓库
2. 在 [vercel.com](https://vercel.com) 导入该仓库
3. Framework Preset 选择 **Other**
4. 点击 Deploy 即可

### 架构说明

| 组件 | 说明 |
|------|------|
| `public/` | 静态HTML，Vercel CDN直接托管 |
| `api/index.py` | Serverless Function，处理 `/api/screen`、`/api/references`、`/api/config` |

部署到Vercel后，AI读图模式的API Key在网页界面中输入，存储在浏览器 localStorage，不会上传到服务器。支持：
- OpenAI（GPT-4o / GPT-4.1）
- Anthropic（Claude）
- 智谱（GLM-4V）
- 通义千问（Qwen-VL）
- DeepSeek

## 十一、常见问题

**Q: 需要联网吗？**

A: 不需要。系统完全本地运行，不调用任何外部API，不上传任何数据。

**Q: 支持手机访问吗？**

A: 支持。Web界面做了响应式适配。在同一局域网内，手机浏览器访问 `http://电脑IP:8000` 即可。

**Q: 系统会替我下单吗？**

A: 不会。系统只做筛查和提醒，所有交易决策和下单由你本人完成。

**Q: 数据从哪来？需要连接券商吗？**

A: 不需要。系统通过你手动填写表单来获取信息，所有判断（市场周期、信号K质量、EMA20位置等）由你从图表上观察后输入。系统的作用是帮你结构化地审视这笔交易，防止遗漏关键检查项。

**Q: 评分准吗？**

A: 评分基于Al Brooks价格行为学的统计概率规则，但任何交易系统都无法保证100%正确。评分高不代表必赚，评分低也不代表必亏。核心价值在于帮你识别违反铁律的交易，避免忙中出错。
